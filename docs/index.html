Gregorian Documentation

System Requirements:

  • Gregorian is made for MODx Evolution, and should work for versions 1.0.*.
  • Gregorian uses some PHP 5 specific features, and will probably not work with older versions of PHP.

Installation

  • Get a copy of Gregorian from http://gregorian.kleistsvendsen.dk and extract it in assets/snippets/Gregorian in your MODx installation.
  • Create config.php as a copy of config.sample.php, and edit the contents to match your database setup.
  • Create module with name "Gregorian" and contents: 
    return include $modx->config['base_path'].'assets/snippets/Gregorian/Gregorian.module.php';
  • Save the module and reload the manager to make the module show up
  • Open the module and click on the "Create database tables"-link. This should produce the following output: 
    Trying to create container for class 'Gregorian'... [Ok]
Trying to create container for class 'GregorianEvent'... [Ok]
Trying to create container for class 'GregorianTag'... [Ok]
Trying to create container for class 'GregorianEventTag'... [Ok]
    You might also get some warnings that the table doesn't exist. They are caused by the way xPDO checks for table existance. If the [Ok] is the result of all 4 attempts, everything should be fine.
  • Create a new snippet with name Gregorian and contents: 
    <?php
return include $modx->config['base_path'].'assets/snippets/Gregorian/Gregorian.snippet.php';
  • Put the snippet call [!Gregorian!] wherever you'd like your calendar to show up.
  • To get the calendar working properly when logged into the manager you have to disable QM+ because of some unresolved javascript conflicts.
